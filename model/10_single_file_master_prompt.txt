# v0.05 v4 Porting Packet for Other LLMs

## Purpose

This packet is for testing whether another LLM can assist with The Model / Remodeled Brain scaffold without collapsing the current v0.05 v4 work into conventional academic defaults, regional folklore, diagnostic-label reasoning, or sycophantic patch adoption.

v0.05 v4 is an ingest-consistency and internal-consistency candidate packet. It is not a bulk corpus revision. It is not a claim-layer rewrite unless explicitly promoted after review.

## Main instruction to the receiving LLM

Treat all v0.05 v4 material in this packet as candidate scaffold work unless the user explicitly says it has been adopted. Your job is to evaluate fit, risks, contradictions, and layer placement before integrating.

Do not automatically rewrite candidate rules as accepted rules.

Do not answer by praise or agreement. Stress-test.

## One-sentence version

v0.05 v4 is not a stronger evidence-weighting layer. It is an admissibility-before-validation layer that prevents folklore handles from becoming causal objects before evidence is evaluated.

# v0.05 v4 Version Goals and Non-Goals

## Status

v0.05 v4 material in this packet is candidate work. It should be evaluated before integration.

## Version goal

Increase paper-ingest rigor and internal consistency by catching folklore, regionalization, proxy-promotion, practical-significance inflation, and spectrum compression before node mapping.

The version should improve the scaffold's ability to ingest papers without allowing author labels or familiar academic frames to enter the model as mechanisms.

## Foundation

The foundation for this version is:

- conserved dependency
- routed maintenance
- demonstrated causal chain
- folklore-stripped residue

## Global shift

Move from:

- rigid regional priority
- brainstem-first as default interpretive route
- anatomical region treated as causal origin
- diagnostic label treated as causal object
- biomarker status treated as mechanism
- statistical significance treated as biological importance

Move to:

- conserved dependency first
- maintenance-route first
- demonstrated causal chain first
- region as implementation site unless source is demonstrated
- diagnostic, biomarker, spectrum, and region labels as handles only
- physical residue before node mapping

## Core principle

Folklore filters fire before validation gates.

Validation evaluates evidentiary strength only after the source generates an admissible physical claim.

## Candidate compact principle

INGEST.PRINCIPLE.05v4:

Ingestion begins by stripping folklore and regional interface text. Only the physically measured substrate, demonstrated variable, causal route, and surviving residue may map to model nodes. Diagnostic labels, region labels, biomarker labels, spectrum labels, and conserved-region labels are handles only. They cannot perform causal work.

## Non-goals

v0.05 v4 does not:

- populate real corpus assessment cells
- perform bulk source walking
- revise the full corpus
- create real corpus weight from fixtures
- allow paper ingest to directly rewrite settled claim nodes
- restore brainstem-first regional priority
- turn conservation into causal weight
- discard papers by default when a gate fails
- replace chain generation with definition sorting
- accept user or external-model patch text by default

## Layer boundary

Paper ingest writes support only unless the user explicitly instructs a claim-layer update and the candidate survives adoption review.

Allowed support-layer outputs include:

- evidence
- provenance
- paper links
- extraction notes
- disconfirmation notes
- proxy-status notes
- category-failure notes
- missing-route notes
- method notes
- source-tree handles
- watch items
- promotion candidates after gates pass
- promotion gate results

Claim-layer writes require gated review.


===== 02_runtime_behavior_contract.md =====

# Runtime Behavior Contract for LLMs

## Purpose

This document tells a receiving LLM how to assist with the v0.05 v4 scaffold without becoming a source of drift.

## Required behavior

When the user supplies a candidate rule, patch, interpretation, or external-model suggestion:

1. Treat it as candidate.
2. Evaluate it before integrating.
3. Separate accept, accept-with-edits, hold, and reject.
4. Identify layer placement.
5. Check contradiction with settled scaffold commitments.
6. Check for reification and hidden operators.
7. Check redundancy with existing rules.
8. Identify failure modes.
9. Provide the smallest safe form.

## Forbidden behavior

Do not:

- log candidate rules as settled without explicit user approval
- expand every suggestion into canonical scaffold text
- praise fit without testing drift
- adopt user phrasing because it matches the direction of the model
- replace chain generation with “is / is not” comparison
- fuse an operation with the product it produces
- use anatomy as causal origin by default
- use diagnostic labels as causal objects
- use biomarker labels as mechanism
- use conservation as prestige or weight
- use statistical significance as biological significance
- use paper polish, sample size, citation count, or publication venue as causal weight

## Known LLM failure modes

### sycophantic adoption

Failure: The LLM treats a candidate rule as accepted because the user proposed it or because it seems directionally aligned.

Repair: Run the Candidate Rule Adoption Gate.

### is-is-not comparator degradation

Failure: The LLM answers by definition sorting: “X is not Y; X is Z.” This replaces generation of the process chain.

Repair: Generate the chain first. Definitions may summarize after the chain.

### operation-product fusion

Failure: The LLM turns a procedure into the object produced by that procedure.

Caught case: “conform” was described as if it were an object-like interaction cut instead of the operation that pressures a claim to generate its interaction-cut path.

Repair: Keep verbs and products separated.

### region-as-source drift

Failure: The LLM treats region activity, perturbability, or conservation as causal origin.

Repair: Region is implementation site unless timing, dependency position, and source status are demonstrated.

### evidence-before-admissibility drift

Failure: The LLM starts weighing a paper before determining whether its causal object survives label stripping.

Repair: Folklore filters must fire before validation gates.

### proxy-as-mechanism drift

Failure: The LLM treats a measured proxy, biomarker, assay readout, or fold change as the mechanism.

Repair: Identify measured substrate, measured variable, proxy status, translation limit, and surviving route.

### conservation-as-prestige drift

Failure: “Highly conserved” is used as an explanation by itself.

Repair: Conservation raises ingest priority only when tied to a conserved operation, dependency relation, or maintenance route.

## Required response shape for evaluating a patch

Use this structure unless the user asks otherwise:

- Verdict: accept / accept with edits / hold / reject.
- Layer: claim layer / support layer / validator / fixture / prose / changelog / workflow.
- Fit: how it supports the current version goal.
- Internal consistency: what it repairs or risks.
- Failure modes: how it could misfire.
- Minimal safe form: the smallest patch text or instruction.
- Adoption status: candidate unless explicitly approved.

INGEST.GATE_FAILURE_ROUTING.05v4:
  status: candidate
  layer: paper_ingest
  purpose: >
    Clarify that ingest gates change what a source may support. They do not discard
    papers by default.

  rule: >
    Gate failure changes what a source may support. It does not remove the source
    from ingest by default. Failed criteria route the source into attenuated support
    channels, method notes, proxy-only evidence, category-failure notes, missing-route
    slots, disconfirmation notes, or source-tree handles. Direct node grounding and
    promotion require gate passage.

  default_failed_gate_result:
    paper_record: keep
    support_layer: write
    direct_node_grounding: deny_or_cap
    promotion_candidate: deny
    weight: attenuate_to_channel_cap

  failure_target:
    - claim_role
    - node_grounding_permission
    - promotion_permission
    - weight_channel
    - translation_status

  preserved_outputs:
    - provenance
    - extraction_notes
    - proxy_status
    - method_notes
    - category_failure_notes
    - missing_route_notes
    - disconfirmation_notes
    - source_tree_handles
    - watch_items

  failed_criterion_routes:
    no_label_stripped_residue:

candidate_ingest_rules_reference:
  status: candidate
  layer: pre_node_mapping
  note: >
    These rules should be evaluated and consolidated before integration. They are
    best treated as a candidate ingest-admissibility cluster rather than many unrelated
    top-level rules.

  v0_05_v4_candidate_ingest_rules:
    purpose: >
      Increase ingest rigor and internal consistency by catching folklore, regionalization,
      proxy-promotion, practical-significance inflation, and spectrum compression before
      node mapping.

    global_shift:
      from:
        - rigid regional priority
        - brainstem-first as default interpretive route
        - anatomical region treated as causal origin
      to:
        - conserved dependency first
        - maintenance-route first
        - demonstrated causal chain first
        - region as implementation site, not explanatory source

    terminology_guard:
      avoid_promoting:
        - brainstem-first
        - cortex-first
        - basal-ganglia controller
        - cerebellum module
        - region X does function Y
        - highly conserved region as standalone explanation
      prefer:
        - conserved maintenance substrate
        - conserved dependency node
        - routed maintenance implementation
        - transition-control surface
        - downstream gating surface
        - stabilization route
        - measured substrate and demonstrated causal route

  LABEL_REMOVAL_RESIDUE_TEST:
    trigger: always_during_paper_ingest
    rule: Remove author labels and ask what physical claim remains.
    strip:
      - diagnosis labels
      - biomarker status
      - disease names as causal objects
      - clinical promise language
      - spectrum generalization
      - controller/command/decision language
      - region-does-function language
      - folklore cognitive labels
      - anatomical prestige language
      - proxy-as-mechanism language
    required_questions:
      - What substrate was measured?
      - What variable changed?
      - What causal route was demonstrated?
      - What claim remains if labels are unavailable?
      - Does the result still map to a model node after stripping?
      - What residue depends on the removed labels?
    failure_effect:
      - no_direct_node_grounding
      - no_promotion_candidate
      - route_to_category_failure_or_extraction_note
    fail_if:
      - no mechanistic claim survives label removal

  FOLKLORE_FILTER_DIAGNOSTIC_LABEL:
    trigger:
      - diagnostic category used as causal object
      - clinical label used to organize measurement
      - disease/syndrome label treated as the thing measured
    rule: >
      Diagnostic labels are handles. They may organize a paper's cohort description,
      but they cannot enter the mechanism as causal objects.
    fail_if:
      - assay cannot generate diagnostic distinction without prior labels
      - label is imported before measurement and then credited as discovered by measurement
      - label substitutes for developmental route
    required_output:
      - measured_substrate
      - measured_variable
      - grouping_variable
      - claim_after_label_stripping
      - residual_label_dependence
    failure_effect:
      - severe_weight_attenuation
      - no_direct_node_grounding
      - support_layer_label_dependent_observation

  REGION_AS_IMPLEMENTATION_NOT_SOURCE:
    trigger:
      - region X controls Y
      - region X generates Y
      - region X is responsible for Y
      - brainstem/cortex/cerebellum/basal-ganglia-first framing
    rule: >
      A region is treated as an implementation site unless the paper demonstrates
      source, dependency position, and causal direction.
    fail_if:
      - activity in a region is treated as origin
      - perturbability is treated as origin
      - downstream deformation is treated as upstream generation
      - conserved-region language substitutes for route evidence
    rewrite_to:
      - measured activity at implementation site
      - perturbation leverage at implementation site
      - causal origin unresolved unless timing/dependency controls close the route
    failure_effect:
      - no_origin_weight
      - implementation_site_observation_only

  CONSERVATION_DEPENDENCY_RULE:
    definition: >
      Conservation increases ingest priority only when the paper identifies a conserved
      operation, dependency, or maintenance route. Conservation alone does not establish
      causal origin.
    pass_if:
      - conserved structure is tied to a demonstrated operation
      - perturbation shows dependency position
      - downstream structures lose explanatory power after conserved-route variables are controlled
      - causal timing runs from conserved maintenance route to later differentiated route
    fail_if:
      - highly conserved is used as prestige language
      - conserved region is named but causal work is assigned to a later folk construct
      - anatomical age is treated as causal sufficiency
      - a late structure is called the controller without controlling upstream conserved-route variables
    rewrite_to:
      - identify the conserved operation
      - identify the dependency relation
      - identify the maintenance route
      - identify the measured causal direction
    invariants:
      conservation_can_prioritize_ingest: true
      conservation_can_supply_weight: false
      conservation_can_supply_origin: false

  CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT:
    trigger:
      - same mutation, different phenotype
      - controlled environment, different outcome
      - same input, divergent result
      - genetically matched or controlled conditions used to imply diagnostic clarity
    rule: >
      Same-input/different-output is not an explanation. It marks the input set as incomplete
      unless the differentiating causal variable is identified.
    fail_if:
      - divergence is renamed as diagnosis
      - divergence is treated as evidence for the category
      - missing variable is not investigated
      - controlled sameness is used to strengthen a diagnostic claim without closing the causal route
    rewrite_to:
      - missing differentiating variable
      - incomplete causal frame
      - divergent developmental stabilization route not identified
    failure_effect:
      - missing_differentiator_slot
      - no_category_clarity_weight

  CONTROLLED_FRAME_INTENSIFIER:
    trigger:
      - controlled assay conditions
      - controlled environment
      - shared mutation
      - matched samples
      - iPSC model described as removing confounds
    rule: >
      Removing environmental variance intensifies the demand to explain remaining divergence.
      It does not license a diagnostic leap.
    fail_if:
      - controlled sameness is treated as explanatory completeness
      - paper uses controlled conditions to imply category specificity
      - remaining divergence is not traced to a causal route
    rewrite_to:
      - controlled frame exposes missing route
      - unexplained divergence means the model's input set is incomplete
    failure_effect:
      - intensified_route_burden
      - missing_route_exposed_by_control

  PRACTICAL_SIGNIFICANCE_GATE:
    trigger:
      - statistically significant
      - analytical significance
      - fold difference
      - nanosensor readout
      - p-value or significant difference language
    rule: >
      Analytical separability does not establish practical, biological, or clinical significance.
    fail_if:
      - no variance envelope
      - no noise floor
      - no effect-size interpretation
      - no biological range
      - no clinical threshold when clinical relevance is claimed
      - no replication across samples, clones, batches, or cohorts
    rewrite_to:
      - assay-level difference only
      - practical significance not established
    failure_effect:
      - assay_level_difference
      - no_biological_or_clinical_weight_without_range_magnitude_replication_and_route

  PROXY_PROMOTION_GATE:
    status: implied_candidate
    rule: >
      A proxy, biomarker, assay readout, or measured correlate cannot become mechanism without
      translation controls and a demonstrated route.
    fail_if:
      - proxy is treated as mechanism
      - biomarker status substitutes for causal route
      - measured variable is promoted beyond its assay level
      - translation limit is not stated
    failure_effect:
      - proxy_only_evidence
      - translation_cap
      - no_direct_node_grounding_if_mechanism_depends_on_proxy_status

  SPECTRUM_COMPRESSION_FILTER:
    status: implied_candidate
    rule: >
      A spectrum label cannot compress heterogeneous developmental or mechanistic routes
      into one causal point.
    fail_if:
      - spectrum is treated as one causal object
      - divergent routes are absorbed into a single diagnostic continuum
      - label supplies unity that measurement did not generate
    failure_effect:
      - category_failure_note
      - route_heterogeneity_note
      - no_single-node_grounding_without_route_split

  CONTROLLER_LANGUAGE_FILTER:
    status: implied_candidate
    rule: >
      Controller, command, decision, and coordination language must be rewritten into
      measured interaction, dependency, route, balance, gating, or release terms before
      mechanism mapping.
    fail_if:
      - controller language supplies hidden operator
      - decision language replaces chain generation
      - command language assigns source without route evidence
    failure_effect:
      - rewrite_required
      - no_mechanism_mapping_until_operator_removed

# v0.05 v4 Fixtures and Regression Tests

## Purpose

These fixtures test whether another LLM can apply v0.05 v4 ingest logic without falling into folklore, regionalization, or sycophantic patch adoption.

Each fixture contains:

- input pattern
- expected filters
- expected allowed support channel
- expected blocked role
- expected rewrite
- common weak-LLM failure

## Fixture 1: Diagnostic label as causal object

Input pattern:

A paper compares autistic and intellectually disabled cohorts using a nanosensor assay. It reports a statistically significant nitric oxide difference and describes NO as a biomarker for autism.

Expected filters:

- FOLKLORE_FILTER_DIAGNOSTIC_LABEL
- LABEL_REMOVAL_RESIDUE_TEST
- PROXY_PROMOTION_GATE
- PRACTICAL_SIGNIFICANCE_GATE

Expected allowed support channel:

- assay-level difference
- label-dependent observation
- proxy-only evidence
- extraction note

Expected blocked role:

- direct autism-node grounding
- biomarker-as-mechanism
- clinical or biological significance unless practical gate passes
- promotion candidate if mechanism disappears after label stripping

Expected rewrite:

The assay measured NO-related signal differences between pre-labeled cohorts. The claim after label stripping is an assay-level group difference. The diagnostic distinction was not generated by the assay. Practical significance and causal route remain unestablished unless magnitude, variance, noise floor, biological range, replication, and route are supplied.

Common weak-LLM failure:

The model says the paper supports autism biomarker discovery because it reports significant differences.

## Fixture 2: Same mutation, different phenotype

Input pattern:

A paper studies the same mutation in genetically similar samples. Some samples show autism-like phenotypes and others show intellectual-disability-like phenotypes. The authors treat this as evidence that the mutation clarifies diagnostic categories.

Expected filters:

- CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT
- CONTROLLED_FRAME_INTENSIFIER if environment or sample background is controlled
- FOLKLORE_FILTER_DIAGNOSTIC_LABEL

Expected allowed support channel:

- missing differentiator slot
- incomplete causal frame note
- developmental route uncertainty

Expected blocked role:

- diagnostic clarity weight
- category grounding
- promotion candidate

Expected rewrite:

Same input with divergent output exposes an incomplete input set. The missing differentiating variable or divergent developmental stabilization route must be identified before the result can ground a diagnostic or model-node claim.

Common weak-LLM failure:

The model says the controlled genetic input strengthens the diagnostic interpretation.

# v0.05 v4 Candidate Changelog

## Status

Candidate. Not adopted unless explicitly approved.

## Summary

v0.05 v4 candidate work shifts paper ingest toward admissibility-before-validation. The model should strip folklore and regional interface text before validation and node mapping.

## Added candidate principle

INGEST.PRINCIPLE.05v4:

Ingestion begins by stripping folklore and regional interface text. Only the physically measured substrate, demonstrated variable, causal route, and surviving residue may map to model nodes. Diagnostic labels, region labels, biomarker labels, spectrum labels, and conserved-region labels are handles only. They cannot perform causal work.

## Added candidate ingest ordering

- strip_author_interface_text
- run_folklore_filters
- run_region_as_implementation_filter
- run_label_removal_residue_test
- run_proxy_promotion_gate
- run_chain_closure_gate
- run_conservation_dependency_gate
- map_surviving_residue_to_nodes
- write_support_layer_only
- mark_promotion_candidates_only_after_gates_pass

## Added candidate rule cluster

- LABEL_REMOVAL_RESIDUE_TEST
- FOLKLORE_FILTER_DIAGNOSTIC_LABEL
- REGION_AS_IMPLEMENTATION_NOT_SOURCE
- CONSERVATION_DEPENDENCY_RULE
- CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT
- CONTROLLED_FRAME_INTENSIFIER
- PRACTICAL_SIGNIFICANCE_GATE
- implied PROXY_PROMOTION_GATE
- implied SPECTRUM_COMPRESSION_FILTER
- implied CONTROLLER_LANGUAGE_FILTER

## Added failure semantics

Gate failure changes the role a source can play. It does not discard papers by default.

Failed gates usually:

- preserve paper record
- allow support-layer writes
- block or cap direct node grounding
- block promotion candidate status
- attenuate weight to the relevant channel

## Added workflow correction

Candidate patches must be evaluated before adoption.

Required checks:

- goal fit
- layer fit
- contradiction check
- reification check
- operation-product check
- chain-generation check
- redundancy check
- failure-mode check
- minimal-patch check

## Known assistant failure corrected

The assistant previously treated user-supplied candidate rules as if they were integrated. This packet corrects that behavior by adding the Patch Adoption Gate and marking v0.05 v4 material as candidate unless explicitly approved.

## Rationale

The NO/autism-style paper fails before evidence weighting when diagnostic labels, biomarker language, diagnosis, and spectrum language are doing causal work the assay did not generate. It can remain in the ingest record as attenuated support, but it cannot directly ground a mechanism or promotion candidate unless label-stripped physical residue and route closure survive.
