# v0.05 v4 Version Goals and Non-Goals

## Status

v0.05 v4 material in this packet is candidate work. It should be evaluated before integration.

## Version goal

Increase paper-ingest rigor and internal consistency by catching folklore, regionalization, proxy-promotion, practical-significance inflation, and spectrum compression before node mapping.

The version should improve the scaffold's ability to ingest papers without allowing author labels or familiar academic frames to enter the model as mechanisms.

## Foundation

The foundation for this version is:

- conserved dependency
- routed maintenance
- demonstrated causal chain
- folklore-stripped residue

## Global shift

Move from:

- rigid regional priority
- brainstem-first as default interpretive route
- anatomical region treated as causal origin
- diagnostic label treated as causal object
- biomarker status treated as mechanism
- statistical significance treated as biological importance

Move to:

- conserved dependency first
- maintenance-route first
- demonstrated causal chain first
- region as implementation site unless source is demonstrated
- diagnostic, biomarker, spectrum, and region labels as handles only
- physical residue before node mapping

## Core principle

Folklore filters fire before validation gates.

Validation evaluates evidentiary strength only after the source generates an admissible physical claim.

## Candidate compact principle

INGEST.PRINCIPLE.05v4:

Ingestion begins by stripping folklore and regional interface text. Only the physically measured substrate, demonstrated variable, causal route, and surviving residue may map to model nodes. Diagnostic labels, region labels, biomarker labels, spectrum labels, and conserved-region labels are handles only. They cannot perform causal work.

## Non-goals

v0.05 v4 does not:

- populate real corpus assessment cells
- perform bulk source walking
- revise the full corpus
- create real corpus weight from fixtures
- allow paper ingest to directly rewrite settled claim nodes
- restore brainstem-first regional priority
- turn conservation into causal weight
- discard papers by default when a gate fails
- replace chain generation with definition sorting
- accept user or external-model patch text by default

## Layer boundary

Paper ingest writes support only unless the user explicitly instructs a claim-layer update and the candidate survives adoption review.

Allowed support-layer outputs include:

- evidence
- provenance
- paper links
- extraction notes
- disconfirmation notes
- proxy-status notes
- category-failure notes
- missing-route notes
- method notes
- source-tree handles
- watch items
- promotion candidates after gates pass
- promotion gate results

Claim-layer writes require gated review.

# Runtime Behavior Contract for LLMs

## Purpose

This document tells a receiving LLM how to assist with the v0.05 v4 scaffold without becoming a source of drift.

## Required behavior

When the user supplies a candidate rule, patch, interpretation, or external-model suggestion:

1. Treat it as candidate.
2. Evaluate it before integrating.
3. Separate accept, accept-with-edits, hold, and reject.
4. Identify layer placement.
5. Check contradiction with settled scaffold commitments.
6. Check for reification and hidden operators.
7. Check redundancy with existing rules.
8. Identify failure modes.
9. Provide the smallest safe form.

## Forbidden behavior

Do not:

- log candidate rules as settled without explicit user approval
- expand every suggestion into canonical scaffold text
- praise fit without testing drift
- adopt user phrasing because it matches the direction of the model
- replace chain generation with “is / is not” comparison
- fuse an operation with the product it produces
- use anatomy as causal origin by default
- use diagnostic labels as causal objects
- use biomarker labels as mechanism
- use conservation as prestige or weight
- use statistical significance as biological significance
- use paper polish, sample size, citation count, or publication venue as causal weight

## Known LLM failure modes

### sycophantic adoption

Failure: The LLM treats a candidate rule as accepted because the user proposed it or because it seems directionally aligned.

Repair: Run the Candidate Rule Adoption Gate.

### is-is-not comparator degradation

Failure: The LLM answers by definition sorting: “X is not Y; X is Z.” This replaces generation of the process chain.

Repair: Generate the chain first. Definitions may summarize after the chain.

### operation-product fusion

Failure: The LLM turns a procedure into the object produced by that procedure.

Caught case: “conform” was described as if it were an object-like interaction cut instead of the operation that pressures a claim to generate its interaction-cut path.

Repair: Keep verbs and products separated.

### region-as-source drift

Failure: The LLM treats region activity, perturbability, or conservation as causal origin.

Repair: Region is implementation site unless timing, dependency position, and source status are demonstrated.

### evidence-before-admissibility drift

Failure: The LLM starts weighing a paper before determining whether its causal object survives label stripping.

Repair: Folklore filters must fire before validation gates.

### proxy-as-mechanism drift

Failure: The LLM treats a measured proxy, biomarker, assay readout, or fold change as the mechanism.

Repair: Identify measured substrate, measured variable, proxy status, translation limit, and surviving route.

### conservation-as-prestige drift

Failure: “Highly conserved” is used as an explanation by itself.

Repair: Conservation raises ingest priority only when tied to a conserved operation, dependency relation, or maintenance route.

## Required response shape for evaluating a patch

Use this structure unless the user asks otherwise:

- Verdict: accept / accept with edits / hold / reject.
- Layer: claim layer / support layer / validator / fixture / prose / changelog / workflow.
- Fit: how it supports the current version goal.
- Internal consistency: what it repairs or risks.
- Failure modes: how it could misfire.
- Minimal safe form: the smallest patch text or instruction.
- Adoption status: candidate unless explicitly approved.

    definition: >

      causal work. Node mapping consumes extracted claim fields, never author labels.

      Author-interface stripping and claim-field extraction (Phase A) run before node mapping.
      Evidence weighting and validation (Phase B) run only after extracted claim fields exist.
      "Folklore filters before validation" is implemented as this A-before-B phase split, not as
      a single ordered slogan.
    note_on_prior_wording: >
      Earlier drafts used "residue" as a structural noun and mapped "surviving residue" to nodes.
      That fused the strip operation with its product. The field is now named post_strip_claim,
      produced in Phase A; only Phase B assigns it a role.

INGEST.GATE_FAILURE_ROUTING.05v4:
  status: candidate
  layer: paper_ingest
  purpose: >
    Clarify that ingest gates change what a source may support. They do not discard papers by
    default. Gate failure caps claim role; it does not delete provenance, descriptive value, or
    method value.

  rule: >
    Gate failure changes what a source may support. It does not remove the source from ingest by
    default. Failed criteria route the source into attenuated support channels, method notes,
    proxy-only evidence, category-failure notes, missing-route slots, disconfirmation notes,
    descriptive-support notes, or source-tree handles. Direct node grounding and promotion require
    gate passage.

  default_failed_gate_result:
    paper_record: keep
    support_layer: write
    direct_node_grounding: deny_or_cap
    promotion_candidate: deny
    weight: attenuate_to_channel_cap

  failure_target:
    - claim_role
    - node_grounding_permission
    - promotion_permission
    - weight_channel
    - translation_status

  # --- v0.05 v4 revision: positive support types on an ordered ladder ---
  support_type_ladder:
    note: >
      Ordered strongest to weakest role. A gate result assigns a level; "not established" means
      "capped at or below the named level," not free discretion. Levels are named, not numeric;
      numeric calibration is out of scope for this candidate (see non-goals in 01).
    levels:
      - mechanistic_grounded
      - route_partial
      - proxy_only
      - descriptive_support
      - label_dependent_observation
      - provenance_only
    level_meanings:
      mechanistic_grounded: closed causal route; survives all Phase B gates
      route_partial: partial route; one or more slots missing
      proxy_only: correlate/biomarker without translation controls
      descriptive_support: valid descriptive/epi/phenomenological/association/first-report
      label_dependent_observation: claim depends on an author handle
      provenance_only: kept for record; no claim role

  over_stripping_guard:
    purpose: >
      Prevent Type-II error: valid descriptive, epidemiological, phenomenological, association, or
      first-report research must not be scored as a failed mechanism paper.
    rule: >
      Absence of a demonstrated mechanism is not a defect. A paper that makes no mechanistic claim,
      but whose post_strip_claim is a sound descriptive/association claim, routes to
      descriptive_support as a positive role, not to a failure note.
    routes:
      descriptive_or_association_claim:
        role: descriptive_support
        not_treated_as: mechanism_failure

  aggregation_limit:
    purpose: Prevent backdoor weight accumulation.
    rule: >
      Support-only and attenuated notes never sum into node grounding by raw count. Node grounding
      requires a passing Phase B gate on a post_strip_claim. Count raises visibility and ingest
      priority only; it never substitutes for gate passage.
    invariants:
      count_can_raise_visibility: true
      count_can_supply_grounding: false
      grounding_requires_passing_gate: true

  gate_intersection_semantics:
    status: candidate
    purpose: Deterministic result when several gates fail on one source; keep support info, cap safely.
    rule: >
      Failed-gate routes accumulate (union of notes; no support-layer information lost), but
      support cap, node-grounding permission, and promotion permission collapse to the most
      restrictive surviving value.
    notes: accumulate
    permissions:
      support_weight: min_across_failed_gates
      node_grounding: denied_if_any_gate_denies
      promotion: denied_if_any_gate_denies
    example:
      failed_gates: [region_source_closure, conservation_dependency, proxy_promotion]
      notes_accumulated: [implementation_site_observation, conservation_handle, proxy_only_evidence]
      result: {causal_weight: none, node_grounding: denied, promotion: denied}

  visibility_priority_cap:
    status: candidate
    purpose: Prevent a large folklore-biased literature from hijacking ingest priority.
    rule: >
      Descriptive volume raises watchlist visibility only. A body of label-dependent,
      zero-closed-route sources cannot gain ingest priority over sources that close a causal
      route, identify a missing differentiator, or disconfirm folklore.
    invariants:
      volume_can_raise_watchlist: true
      volume_can_outrank_route_closing_sources: false
      correlated_sources_count_once: true    # shared pipeline / lab / cohort are not independent
    deferred:
      numeric_scaling: out_of_scope_until_real_corpus
      note: >
        log scaling, absolute caps, and label/route/proxy multipliers need calibration data;
        non-goal 01 forbids creating corpus weight from fixtures, so this stays qualitative.

  preserved_outputs:
    - provenance
    - extraction_notes
    - proxy_status
    - method_notes
    - category_failure_notes
    - missing_route_notes
    - disconfirmation_notes
    - descriptive_support_notes
    - source_tree_handles
    - watch_items

  failed_criterion_routes:
    no_post_strip_claim:
      route_to:
        - category_failure_note
        - extraction_note
      weight_effect: no_direct_node_grounding

    diagnostic_label_required_for_claim:
      route_to:
        - label_dependent_observation
        - cohort_grouping_note
      weight_effect: severe_attenuation

    proxy_promoted_to_mechanism:

        - proxy_only_evidence

    practical_significance_not_established:
      route_to:
        - assay_level_difference
        - magnitude_or_noise_floor_missing
      weight_effect: no_biological_or_clinical_weight

    controlled_sameness_missing_differentiator:
      route_to:
        - missing_differentiator_slot

      weight_effect: no_category_clarity_weight
      note: merged route for former same_input_different_output + controlled_frame_divergence

    region_as_source_unclosed:

    conservation_without_dependency:

  hard_zero_scope:
    note: >
      Hard zero applies to claim ROLE, never to the paper. It zeroes specific role permissions

    folklore_filters_before_validation:
      effect: classify_and_cap
      not_effect: discard_by_default

candidate_ingest_rules_reference:
  status: candidate
  layer: pre_node_mapping
  note: >
    Consolidated for v0.05 v4. The former six label-family filters are now one STRIP_AND_EXTRACT
    gate (Phase A) plus a handle-type table. The two controlled-sameness rules are merged. Route
    and evidence gates (proxy, practical-significance, region-source, conservation-dependency) run
    in Phase B on already-extracted claim fields. This remains a candidate ingest-admissibility
    cluster, not adopted rules.

  v0_05_v4_candidate_ingest_rules:
    purpose: >
      Increase ingest rigor and internal consistency by catching folklore, regionalization,
      proxy-promotion, practical-significance inflation, and spectrum compression before node
      mapping.

    global_shift:
      from:
        - rigid regional priority
        - brainstem-first as default interpretive route
        - anatomical region treated as causal origin
      to:
        - conserved dependency first
        - maintenance-route first
        - demonstrated causal chain first
        - region as implementation site, not explanatory source

    terminology_guard:
      avoid_promoting:
        - brainstem-first
        - cortex-first
        - basal-ganglia controller
        - cerebellum module
        - region X does function Y
        - highly conserved region as standalone explanation
      prefer:
        - conserved maintenance substrate
        - conserved dependency node
        - routed maintenance implementation
        - transition-control surface
        - downstream gating surface
        - stabilization route
        - measured substrate and demonstrated causal route
      caution: >
        The "prefer" terms are descriptive handles, not causal objects. They name measured
        interactions or routes; they must not become seats, controllers, or essences. The same
        verb/product discipline as post_strip_claim applies (see reification_note).

    reification_note: >
      Do not treat a produced field as an object with causal powers. post_strip_claim (formerly
      "residue") is the PRODUCT of the strip-and-extract operation; it is mapped to nodes only
      after Phase B assigns it a role. Keep the operation (strip/extract) and its product
      (post_strip_claim) separate.

  post_strip_claim_form:
    status: candidate
    purpose: >
      Force support-bearing claims into physical slots so a natural-language post_strip_claim
      cannot launder a hidden causal operator after labels are removed. Slottability is the
      structural admissibility test: if a claim cannot be slotted, it cannot ground or promote.
    slot_types:
      - observation
      - causal_relation
      - proxy_relation
      - category_failure
    causal_relation_minimum:
      # trimmed from the full 8-field form; expand only if fixtures require it
      - substrate_A
      - direction
      - substrate_B
      - unresolved_route_slots        # explicit list of what is not yet closed
    prose_location: extraction_notes   # free-text explanation stays here, not in the slot
    guard: slot assignment is driven by what was measured, not by the author label

  # ---------------- PHASE A: one gate + handle-type table ----------------
  STRIP_AND_EXTRACT_GATE:
    phase: A_admissibility
    trigger: always_during_paper_ingest
    consolidates:
      - LABEL_REMOVAL_RESIDUE_TEST
      - FOLKLORE_FILTER_DIAGNOSTIC_LABEL
      - SPECTRUM_COMPRESSION_FILTER
      - CONTROLLER_LANGUAGE_FILTER
      - BIOMARKER_STATUS_FILTER
      - REGION_LABEL_FILTER
    rule: >
      Remove author handles and ask what physical claim remains, then extract claim fields. This is
      structural: it does not weigh evidence.
    required_questions:
      - What substrate was measured?
      - What variable changed?
      - What is the grouping variable?
      - What claim remains if handles are unavailable? (post_strip_claim)
      - What depends on the removed handles? (handle_dependencies)
    output_fields:
      - measured_substrate
      - measured_variable
      - grouping_variable
      - post_strip_claim
      - handle_dependencies
    handle_types:
      diagnostic_label:
        is_handle_for: cohort description / grouping
        cannot: enter mechanism as a causal object
        fail_if:
          - assay cannot generate the diagnostic distinction without prior labels
          - label imported before measurement then credited as discovered by measurement
          - label substitutes for developmental route
      spectrum_label:
        cannot: compress heterogeneous routes into one causal point
        fail_if:
          - spectrum treated as one causal object
          - divergent routes absorbed into a single diagnostic continuum
          - label supplies unity measurement did not generate
      controller_command_language:
        cannot: supply a hidden operator / decision / command source
        rewrite_to: measured interaction, dependency, route, balance, gating, release
        fail_if:
          - controller language supplies hidden operator
          - decision language replaces chain generation
      biomarker_status:
        cannot: substitute for a causal route
        fail_if:
          - biomarker status treated as mechanism
      region_label:
        cannot: supply causal origin by name / activity / perturbability
        rewrite_to: measured activity or perturbation leverage at an implementation site
        fail_if:
          - region-does-function language used as origin
        note: source status is decided in Phase B (REGION_SOURCE_CLOSURE), not here
      conservation_label:
        cannot: act as prestige or standalone explanation
        fail_if:
          - '"highly conserved" used as explanation by itself'
        note: dependency/route is decided in Phase B (CONSERVATION_DEPENDENCY)
      clinical_promise_language:
        cannot: supply biological or clinical weight
        fail_if:
          - clinical promise substitutes for a demonstrated route
    admissibility_verdict:
      admissible_if: a post_strip_claim exists
      inadmissible_if: no physical claim survives handle removal
    failure_effect:
      - no_direct_node_grounding
      - no_promotion_candidate
      - route_to_category_failure_or_extraction_note
      - if a sound descriptive claim survives, route_to descriptive_support (not failure)

  # ---------------- PHASE B: route / evidence gates on extracted fields ----------------
  CONTROLLED_SAMENESS_MISSING_DIFFERENTIATOR:
    phase: B_validation
    merges:
      - CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT
      - CONTROLLED_FRAME_INTENSIFIER
    trigger:
      - same mutation, different phenotype
      - controlled/matched conditions, different outcome
      - iPSC or controlled assay described as removing confounds
      - controlled sameness used to imply diagnostic clarity
    rule: >
      Removing environmental variance intensifies the demand to explain remaining divergence; it
      does not license a diagnostic leap. Same-input/different-output marks the input set as
      incomplete until the differentiating causal variable is identified.
    fail_if:
      - divergence renamed as diagnosis or treated as evidence for the category
      - missing differentiating variable not investigated
      - controlled sameness treated as explanatory completeness
    rewrite_to:
      - missing differentiating variable
      - incomplete causal frame
      - controlled frame exposes missing route
    failure_effect:
      - missing_differentiator_slot
      - no_category_clarity_weight
      - intensified_route_burden

  REGION_SOURCE_CLOSURE:
    phase: B_validation
    rule: >
      A region is an implementation site unless the paper demonstrates source, dependency position,
      and causal direction. Can interact with CONSERVATION_DEPENDENCY; both run on extracted fields
      before roles are assigned.
    fail_if:
      - activity / perturbability / downstream deformation treated as upstream generation
      - conserved-region language substitutes for route evidence
    rewrite_to:
      - measured activity at implementation site
      - perturbation leverage at implementation site
      - causal origin unresolved unless timing/dependency controls close the route
    failure_effect:
      - no_origin_weight
      - implementation_site_observation_only

  CONSERVATION_DEPENDENCY:
    phase: B_validation
    definition: >
      Conservation increases ingest priority only when the paper identifies a conserved operation,
      dependency, or maintenance route. Conservation alone does not establish causal origin or
      weight.
    pass_if:
      - conserved structure tied to a demonstrated operation
      - perturbation shows dependency position
      - downstream structures lose explanatory power after conserved-route variables are controlled
      - causal timing runs from conserved maintenance route to later differentiated route
    fail_if:
      - '"highly conserved" used as prestige language'
      - conserved region named but causal work assigned to a later folk construct
      - anatomical age treated as causal sufficiency
      - a late structure called controller without controlling upstream conserved-route variables
    invariants:
      conservation_can_prioritize_ingest: true
      conservation_can_supply_weight: false
      conservation_can_supply_origin: false

  PROXY_PROMOTION_GATE:
    phase: B_validation
    rule: >
      A proxy, biomarker, assay readout, or measured correlate cannot become mechanism without
      translation controls and a demonstrated route. Distinct from PRACTICAL_SIGNIFICANCE_GATE:
      this gate is about mechanistic translation, not effect magnitude.
    fail_if:
      - proxy treated as mechanism
      - biomarker status substitutes for causal route
      - measured variable promoted beyond its assay level
      - translation limit not stated
    shared_output_with_practical_significance: assay_level_cap
    failure_effect:
      - proxy_only_evidence
      - translation_cap
      - no_direct_node_grounding_if_mechanism_depends_on_proxy_status

  PRACTICAL_SIGNIFICANCE_GATE:
    phase: B_validation
    rule: >
      Analytical separability does not establish practical, biological, or clinical significance.
      Distinct from PROXY_PROMOTION_GATE: this gate is about effect magnitude and replication, not
      mechanistic translation.
    fail_if:
      - no variance envelope
      - no noise floor
      - no effect-size interpretation
      - no biological range
      - no clinical threshold when clinical relevance is claimed
      - no replication across samples, clones, batches, or cohorts
    shared_output_with_proxy_promotion: assay_level_cap
    failure_effect:
      - assay_level_difference
      - no_biological_or_clinical_weight_without_range_magnitude_replication_and_route

# v0.05 v4 Fixtures and Regression Tests

## Purpose

These fixtures test whether another LLM can apply v0.05 v4 ingest logic without falling into folklore, regionalization, or sycophantic patch adoption.

Each fixture contains:

- input pattern
- expected filters
- expected allowed support channel
- expected blocked role
- expected rewrite
- common weak-LLM failure

## Rule-name legend (post-consolidation)

The v0.05 v4 consolidation replaced several per-label filters with one Phase-A gate plus a handle-type table, and merged the two controlled-sameness rules. Fixtures below use the consolidated names:

- `STRIP_AND_EXTRACT_GATE` (Phase A) — subsumes the former `LABEL_REMOVAL_RESIDUE_TEST`, `FOLKLORE_FILTER_DIAGNOSTIC_LABEL`, `SPECTRUM_COMPRESSION_FILTER`, `CONTROLLER_LANGUAGE_FILTER`, biomarker filter, and region-label stripping. Named handle types: `diagnostic_label`, `spectrum_label`, `controller_command_language`, `biomarker_status`, `region_label`, `conservation_label`, `clinical_promise_language`.
- `CONTROLLED_SAMENESS_MISSING_DIFFERENTIATOR` (Phase B) — merges the former `CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT` and `CONTROLLED_FRAME_INTENSIFIER`.
- `REGION_SOURCE_CLOSURE` (Phase B) — the source/dependency-closure half of the former `REGION_AS_IMPLEMENTATION_NOT_SOURCE` (the label-stripping half is the `region_label` handle).
- `CONSERVATION_DEPENDENCY` (Phase B) — former `CONSERVATION_DEPENDENCY_RULE`.
- `PROXY_PROMOTION_GATE`, `PRACTICAL_SIGNIFICANCE_GATE`, `PATCH_ADOPTION_GATE` — unchanged.

## Fixture 1: Diagnostic label as causal object

Input pattern:

A paper compares autistic and intellectually disabled cohorts using a nanosensor assay. It reports a statistically significant nitric oxide difference and describes NO as a biomarker for autism.

Expected filters:

- STRIP_AND_EXTRACT_GATE (handle: diagnostic_label, biomarker_status)
- PROXY_PROMOTION_GATE
- PRACTICAL_SIGNIFICANCE_GATE

Expected allowed support channel:

- assay-level difference
- label-dependent observation
- proxy-only evidence
- extraction note

Expected blocked role:

- direct autism-node grounding
- biomarker-as-mechanism
- clinical or biological significance unless practical gate passes
- promotion candidate if mechanism disappears after label stripping

Expected rewrite:

The assay measured NO-related signal differences between pre-labeled cohorts. The claim after label stripping is an assay-level group difference. The diagnostic distinction was not generated by the assay. Practical significance and causal route remain unestablished unless magnitude, variance, noise floor, biological range, replication, and route are supplied.

Common weak-LLM failure:

The model says the paper supports autism biomarker discovery because it reports significant differences.

## Fixture 2: Same mutation, different phenotype

Input pattern:

A paper studies the same mutation in genetically similar samples. Some samples show autism-like phenotypes and others show intellectual-disability-like phenotypes. The authors treat this as evidence that the mutation clarifies diagnostic categories.

Expected filters:

- CONTROLLED_SAMENESS_MISSING_DIFFERENTIATOR
- STRIP_AND_EXTRACT_GATE (handle: diagnostic_label)

Expected allowed support channel:

- missing differentiator slot
- incomplete causal frame note
- developmental route uncertainty

Expected blocked role:

- diagnostic clarity weight
- category grounding
- promotion candidate

Expected rewrite:

Same input with divergent output exposes an incomplete input set. The missing differentiating variable or divergent developmental stabilization route must be identified before the result can ground a diagnostic or model-node claim.

Common weak-LLM failure:

The model says the controlled genetic input strengthens the diagnostic interpretation.

## Fixture 3: Controlled iPSC frame with remaining divergence

Input pattern:

An iPSC model is described as removing environmental confounds. The paper reports divergent outcomes across clones or lines and treats the controlled frame as making the disease category more specific.

Expected filters:

- CONTROLLED_SAMENESS_MISSING_DIFFERENTIATOR
- PRACTICAL_SIGNIFICANCE_GATE if differences are statistical only

Expected allowed support channel:

- controlled-frame missing-route note
- clone/batch/sample divergence note
- method note

Expected blocked role:

- diagnostic specificity
- causal closure
- direct node grounding

Expected rewrite:

The controlled frame increases the burden to explain remaining divergence. It does not supply causal completeness.

Common weak-LLM failure:

The model treats reduced environmental confounds as sufficient to make the remaining difference disease-specific.

## Fixture 4: Statistical significance without practical significance

Input pattern:

A paper reports a tiny but statistically significant group difference using a sensitive assay. It does not report noise floor, variance envelope, biological range, or functional threshold.

Expected filters:

- PRACTICAL_SIGNIFICANCE_GATE
- PROXY_PROMOTION_GATE if the assay output is promoted

Expected allowed support channel:

- assay-level difference
- practical significance not established
- effect-size limitation note

Expected blocked role:

- biological significance
- clinical significance
- mechanism weight
- promotion candidate

Expected rewrite:

The assay separated the reported values under the study conditions. Biological and clinical significance remain unestablished.

Common weak-LLM failure:

The model treats p-value or sensitivity as biological importance.

## Fixture 5: Region activity treated as origin

Input pattern:

A paper finds cortical activity during a behavior and states that cortex controls the behavior. It does not control upstream conserved-route variables or demonstrate timing and dependency position.

Expected filters:

- STRIP_AND_EXTRACT_GATE (handle: region_label, controller_command_language)
- REGION_SOURCE_CLOSURE

Expected allowed support channel:

- implementation-site observation
- activity-location note
- route-unresolved note

Expected blocked role:

- cortical origin claim
- controller claim
- direct source mapping

Expected rewrite:

The paper measured activity at an implementation site. Source status remains unresolved unless timing, dependency, and upstream-route controls close the route.

Common weak-LLM failure:

The model maps the function to cortex because the activity is located there.

## Fixture 6: Conserved region treated as causal sufficiency

Input pattern:

A paper says a highly conserved brainstem region explains a behavior because the region is ancient and involved in the behavior.

Expected filters:

- CONSERVATION_DEPENDENCY
- REGION_SOURCE_CLOSURE

Expected blocked role:

- mechanism weight from conservation alone

Common weak-LLM failure:

The model treats “highly conserved” as proof that the region is the source.

## Fixture 7: Biomarker/proxy promoted to mechanism

Expected blocked role:

Common weak-LLM failure:

The model treats “biomarker” as causal mechanism.

## Fixture 8: Spectrum compression

Input pattern:

A paper reports differences across an autism spectrum sample and treats the spectrum as one explanatory object.

Common weak-LLM failure:

The model boosts the majority claim because many papers report it, and treats repetition as evidentiary weight.

# v0.05 v4 Candidate Changelog

## Status

Candidate. Not adopted unless explicitly approved.

## Summary

v0.05 v4 candidate work shifts paper ingest toward admissibility-before-validation. The model should strip folklore and regional interface text before validation and node mapping.

## Added candidate principle

INGEST.PRINCIPLE.05v4:

Ingestion begins by stripping folklore and regional interface text. Only the physically measured substrate, demonstrated variable, causal route, and surviving residue may map to model nodes. Diagnostic labels, region labels, biomarker labels, spectrum labels, and conserved-region labels are handles only. They cannot perform causal work.

## Added candidate ingest ordering

- strip_author_interface_text
- run_folklore_filters
- run_region_as_implementation_filter
- run_label_removal_residue_test
- run_proxy_promotion_gate
- run_chain_closure_gate
- run_conservation_dependency_gate
- map_surviving_residue_to_nodes
- write_support_layer_only
- mark_promotion_candidates_only_after_gates_pass

## Added candidate rule cluster

- LABEL_REMOVAL_RESIDUE_TEST
- FOLKLORE_FILTER_DIAGNOSTIC_LABEL
- REGION_AS_IMPLEMENTATION_NOT_SOURCE
- CONSERVATION_DEPENDENCY_RULE
- CHAIN_BREAK_SAME_INPUT_DIFFERENT_OUTPUT
- CONTROLLED_FRAME_INTENSIFIER
- PRACTICAL_SIGNIFICANCE_GATE
- implied PROXY_PROMOTION_GATE
- implied SPECTRUM_COMPRESSION_FILTER
- implied CONTROLLER_LANGUAGE_FILTER

## Added failure semantics

Gate failure changes the role a source can play. It does not discard papers by default.

Failed gates usually:

- preserve paper record
- allow support-layer writes
- block or cap direct node grounding
- block promotion candidate status
- attenuate weight to the relevant channel

## Added workflow correction

Candidate patches must be evaluated before adoption.

Required checks:

- goal fit
- layer fit
- contradiction check
- reification check
- operation-product check
- chain-generation check
- redundancy check
- failure-mode check
- minimal-patch check

## Known assistant failure corrected

The assistant previously treated user-supplied candidate rules as if they were integrated. This packet corrects that behavior by adding the Patch Adoption Gate and marking v0.05 v4 material as candidate unless explicitly approved.

## Rationale

The NO/autism-style paper fails before evidence weighting when diagnostic labels, biomarker language, diagnosis, and spectrum language are doing causal work the assay did not generate. It can remain in the ingest record as attenuated support, but it cannot directly ground a mechanism or promotion candidate unless label-stripped physical residue and route closure survive.
