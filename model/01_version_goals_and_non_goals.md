# v0.05 v4 Version Goals and Non-Goals

## Status

v0.05 v4 material in this packet is candidate work. It should be evaluated before integration.

## Version goal

Increase paper-ingest rigor and internal consistency by catching folklore, regionalization, proxy-promotion, practical-significance inflation, and spectrum compression before node mapping.

The version should improve the scaffold's ability to ingest papers without allowing author labels or familiar academic frames to enter the model as mechanisms.

## Foundation

The foundation for this version is:

- conserved dependency
- routed maintenance
- demonstrated causal chain
- the folklore-stripped physical claim (`post_strip_claim`)

## Global shift

Move from:

- rigid regional priority
- brainstem-first as default interpretive route
- anatomical region treated as causal origin
- diagnostic label treated as causal object
- biomarker status treated as mechanism
- statistical significance treated as biological importance

Move to:

- conserved dependency first
- maintenance-route first
- demonstrated causal chain first
- region as implementation site unless source is demonstrated
- diagnostic, biomarker, spectrum, and region labels as handles only
- `post_strip_claim` before node mapping

## Core principle

Folklore filters fire before validation gates.

Validation evaluates evidentiary strength only after the source generates an admissible physical claim.

## Candidate compact principle

INGEST.PRINCIPLE.05v4:

Ingest runs in two phases. Phase A (admissibility) strips folklore and regional interface text and extracts claim fields — measured substrate, measured variable, grouping variable, and the `post_strip_claim` that remains once labels are treated as handles. Phase B (validation) then weights evidence and assigns a role. Only the extracted `post_strip_claim` may map to model nodes, and only after Phase B. Diagnostic labels, region labels, biomarker labels, spectrum labels, and conserved-region labels are handles only. They cannot perform causal work.

## Non-goals

v0.05 v4 does not:

- populate real corpus assessment cells
- perform bulk source walking
- revise the full corpus
- create real corpus weight from fixtures
- allow paper ingest to directly rewrite settled claim nodes
- restore brainstem-first regional priority
- turn conservation into causal weight
- discard papers by default when a gate fails
- replace chain generation with definition sorting
- accept user or external-model patch text by default

## Layer boundary

Paper ingest writes support only unless the user explicitly instructs a claim-layer update and the candidate survives adoption review.

Allowed support-layer outputs include:

- evidence
- provenance
- paper links
- extraction notes
- disconfirmation notes
- proxy-status notes
- category-failure notes
- missing-route notes
- method notes
- source-tree handles
- watch items
- promotion candidates after gates pass
- promotion gate results

Claim-layer writes require gated review.
