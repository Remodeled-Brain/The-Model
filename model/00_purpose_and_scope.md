# The Model — Purpose and Scope

**Status:** CANDIDATE architectural refactor.

## Primary purpose

The Model reconstructs questions expressed through folklore, reified categories, hidden operators, or incomplete causal language into questions the empirical chain can answer.

The runtime must:

1. preserve the observation, distinction, experience, or requested relation carried by the source question;
2. identify labels and explanatory handles that cannot perform causal work;
3. reconstruct the question in physical substrates, variables, operations, dependencies, timing, constraints, scales, and organism-state transitions;
4. retain multiple candidate routes when the observation does not select one;
5. identify the measurements and discriminating observations required to close those routes;
6. bind reviewed evidence to the exact chain segments and joins it supports;
7. return the strongest bounded answer supported by the closed path; and
8. translate the result back into language responsive to the source question without restoring removed handles as mechanism.

## Operating chain

`source question -> observation preservation -> physical reconstruction -> route generation -> answerability plan -> evidence binding -> bounded answer -> responsive translation`

## Adaptation

Paper ingest is the evidence-maintenance subsystem under the runtime. It keeps the corpus current, provenance-bound, and resistant to folklore, proxy promotion, route breaks, scale jumps, and recency-biased framing.

The primary runtime remains operable against a frozen reviewed corpus. New ingest may alter available evidence, supersession status, segment closure, disconfirmation, and conclusion-specific confidence. It may not silently alter the chain kernel, question-compilation grammar, or settled claim layer.

## Confidence

Confidence remains attached to the operation that generated it:

- **compilation confidence:** security of the mapping from the source question to physical variables and candidate routes;
- **segment confidence:** closure of each required chain segment and join;
- **scale-translation confidence:** closure of the physical path between measured and requested scales;
- **answer-translation confidence:** how directly the closed chain answers the source wording.

These dimensions remain separate. Each conclusion carries confidence from the path it actually requires.

## Non-goals

The Model does not:

- treat scientific labels as causal objects;
- answer by definition sorting in place of chain generation;
- require live paper ingest to operate;
- promote recency, publication venue, citation count, or framework agreement into causal weight;
- erase observations when their customary explanation fails;
- force one route when several remain compatible with the evidence; or
- allow evidence maintenance to rewrite settled runtime commitments without explicit adoption.
